## 2. Picker — новый flow

Подробно зафиксировано в **ADR-003** (`docs/decisions/ADR-003-picker-item-state-machine.md`).

**4 состояния позиции:** Необработано → Собрано / Отсутствует → Заменено

| Состояние | Смысл |
|-----------|-------|
| Необработано | qty=0, ещё не обработал |
| Собрано | qty > 0 |
| Отсутствует | явно отметил отсутствующим, замены нет |
| Заменено | отсутствует + замена связана в UI-state |

**Guards завершения:**
1. Есть Необработанные → блок
2. Нет ни одного Собрано/Заменено → блок
3. REPLACE стратегия + есть Отсутствующие без замены → блок

**Стратегии:**
- `REMOVE` (AUTO/CALL) — удаление свободно, замена опциональна (бэкдор)
- `REPLACE` (AUTO/CALL) — удаление без замены заблокировано
- `CALL_*` — `tel:` кнопка в шапке, звонок необязателен технически

**Изменения в коде:**
- `UpdateOrderItemsUseCase` — разрешить qty=0 (позиция с qty=0 исключается из newItems)
- `CompletePickingUseCase` — три guards выше
- `PickingWorkspace` — четыре группы, кнопка «Отсутствует», restore, tel:

**Отложено:** product search для замены, подтверждение звонка, частичное отсутствие
